# GitLab CI

The CLI expects a JSON manifest that can be generated by CI before review.

```yaml
stages:
  - review

infra-review:
  stage: review
  image: python:3.12-slim
  script:
    - python -m toolkit.cli review "$MR_MANIFEST" --pretty --fail-on high
  artifacts:
    when: always
    paths:
      - review-report.json
```

For production usage, keep the manifest generation step close to the source
system. The review engine only needs changed file content and optional desired
versus observed state snapshots.
